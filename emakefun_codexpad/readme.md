# CodexPad 蓝牙手柄

CodexPad 蓝牙游戏手柄 Blockly 库，支持通过 BLE 连接 CodexPad 手柄并实时读取按键和摇杆输入。

## 库信息

- **库名**: @aily-project/lib-codexpad
- **版本**: 1.0.0
- **兼容**: ESP32 全系列（ESP32、ESP32-S2、ESP32-S3、ESP32-C3、ESP32-C6 等）
- **原始库**: [codex_pad_arduino_lib](https://github.com/CodexPad/codex_pad_arduino_lib)

## 功能特性

- 通过蓝牙 MAC 地址连接 CodexPad 手柄
- 实时按键事件检测：按下、释放、持续按下、当前状态
- 高精度摇杆数据读取（0~255）
- 可调蓝牙发射功率（-16dBm ~ +6dBm）

## 使用前准备

1. **获取手柄 MAC 地址**：参考 CodexPad 产品手册获取手柄的蓝牙 MAC 地址（格式如 `E4:66:E5:A2:24:5D`）
2. **开启手柄**：将手柄开机，确保指示灯呈慢闪状态（待连接状态）

## 块说明

### 初始化

| 块 | 说明 |
|---|---|
| **初始化CodexPad手柄** | 创建手柄对象并连接到指定 MAC 地址，自动在主循环中更新数据 |
| **已连接** | 检查手柄是否已连接，返回布尔值 |
| **设置发射功率** | 调整蓝牙发射功率，影响通信距离和功耗 |

### 按键检测

| 块 | 说明 |
|---|---|
| **按键刚按下** | 检测按键从弹起变为按下的瞬间 |
| **按键刚释放** | 检测按键从按下变为弹起的瞬间 |
| **按键持续按下** | 检测按键是否一直被按住 |
| **按键状态** | 获取按键当前状态（按下/弹起） |

### 摇杆

| 块 | 说明 |
|---|---|
| **摇杆值** | 获取摇杆轴值，范围 0~255，中心约 128 |
| **摇杆值变化** | 检测摇杆是否发生超过阈值的显著变化 |

## 可用按键

上、下、左、右、X/□、Y/△、A/×、B/○、L1、L2、L3、R1、R2、R3、Select、Start、Home

## 可用摇杆轴

左摇杆X轴、左摇杆Y轴、右摇杆X轴、右摇杆Y轴

## 注意事项

1. **仅支持 ESP32**：本库依赖 ESP32 BLE 功能，不支持 Arduino AVR 等平台
2. **MAC 地址必填**：初始化时必须填入手柄的实际 MAC 地址
3. **实时更新**：`Update()` 已自动添加到主循环，无需手动调用
4. **无延时要求**：主循环中应避免使用 `delay()`，以确保手柄数据及时更新
