# OpenJumper 2.4G无线遥控器数据解析库

用于解析OpenJumper 2.4G无线遥控器数据的Blockly库，支持读取手柄摇杆、按键数据。
注意：在使用该库时，需要同时加载ArduinoJSON库

## 特性

-简单易用: 提供直观的积木块接口，无需复杂编程
-快速发送: 数据通过2.4G模组发送及接收，速度快丢包率低
-数据标准: 使用JSON的数据格式进行传输，兼容性高

## 支持的开发板

- Arduino UNO/Nano

## 硬件连接

### 标准连接方式（串口）
| 接收模块引脚 | Arduino UNO |
|-------------|-------------|
| TX          | 0           |
| RX          | 1           |
| -           | GND         |
| 5V          | 5V          |

## 功能积木

### 初始化
- **初始化数据解析**: 配置JSON数据解析

### 数据持续解析
- **持续对接收到的数据进行解析**: 无返回值

### 判断是否接收到数据
- **判断是否接收到数据**: 接收到数据返回1（真），未接收到返回0（假）

### 数据操作
- **按键逻辑**: 返回每个按键的逻辑数据，布尔值，包括摇杆上的两个按键
- **摇杆数据**: 返回摇杆的模拟数据，值在0~255，中间值是127

## 使用示例

### 基础卡片检测
```
初始化无线手柄 连接串口
保持无线手柄数据接收及解析
如果 接收到数据 执行
  读取手柄按键或摇杆数据
```

## 技术规格

- **通信接口**: 串口
- **工作电压**: 5V

## 注意事项

1. **电压兼容**: 无线手柄数据接收模块工作电压为5V，使用3.3V开发板时需要电平转换
2. **接线正确**: 确保串口接线正确，特别是引脚不要接反
3. **连接开关**: 无线手柄数据接收模块上有一个串口连接开关，使用时需要拨动到ON一端
4. **数据格式**: 串口中数据格式为JSON数据

## 依赖库

本库基于以下开源库构建：
- **ArduinoJson** - JSON库
另外，使用了一个自建库：
- **OpenJumper_RemoteControl** - 手柄数据解析库
